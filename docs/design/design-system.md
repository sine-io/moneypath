# UI/UX 设计系统

## 1. 设计原则

- **新手友好**：默认不暴露过多金融术语。
- **游戏化**：进度、徽章、关卡感。
- **低压力**：避免红绿过强刺激，降低恐慌感。
- **解释优先**：所有图表都要有自然语言解释。
- **移动优先**：大部分用户会先在手机上浏览。

## 2. 视觉风格

### 2.1 色彩系统

| 名称 | 用途 | 色值 |
|---|---|---|
| Primary | 主按钮、关键高亮 | `#4F46E5` |
| Success | 正向收益、完成态 | `#16A34A` |
| Warning | 风险提示、中性提醒 | `#F59E0B` |
| Danger | 严重风险、错误 | `#DC2626` |
| Background | 页面背景 | `#0B1020` 或浅色主题 |
| Surface | 卡片底色 | `#111827` / `#FFFFFF` |

### 2.2 字体

- 标题：Inter / Noto Sans SC
- 正文：Inter / 思源黑体
- 数字：等宽数字优先，便于阅读收益曲线

## 3. 关键页面规范

### 3.1 首页

信息结构：
1. 一句话价值主张。
2. 三大功能入口：学习、模拟、AI。
3. 新手收益示例（不是承诺收益）。
4. 免责声明。

### 3.2 课程页

页面分区：
- 顶部：课程标题、关卡进度、预计时长。
- 中间：图文内容块。
- 侧边：术语卡、知识提示。
- 底部：互动题和测验。

### 3.3 模拟交易页

页面分区：
- 左侧：账户总览。
- 中间：持仓列表、交易历史。
- 右侧：买卖面板、风险提示。

### 3.4 AI导师页

页面分区：
- 中间主区域：聊天窗口。
- 右侧辅助区：常见问题、快捷提问。
- 输入框支持语音转文字、截图上传。

## 5. 交互规范

- 所有重要操作必须二次确认。
- 交易按钮在资金不足时禁用。
- 测验答错后给出解释，不只给对错。
- AI回答太长时自动折叠，并提供“展开详情”。
- 图表 hover 时显示数值和说明。

## 6. 适配规范

### 桌面端
- 左右三栏布局。
- 适合分析和对比。

### 移动端
- 单栏滚动。
- 底部固定导航：学习 / 模拟 / AI / 工具 / 我的。

## 7. 无障碍

- 所有按钮有 aria-label。
- 颜色不是唯一信息载体。
- 图表需提供文字摘要。
- 字体大小至少 14px。

## 8. 文案语气

- 友好、明确、低威压。
- 避免“你必须”“你应该很快赚钱”这种措辞。
- 统一使用“学习”“模拟”“练习”“风险提示”等中性词。
